Introduction to HTML Media

HTML5 introduced powerful multimedia elements that allow you to embed audio, video, and graphics directly into your web pages without plugins. This tutorial covers the main multimedia elements: <audio>, <video>, and <svg>.

üìù Pure HTML Focus

This tutorial focuses exclusively on HTML elements and attributes. We won't cover CSS styling or JavaScript interactions - those will be covered in separate CSS and JavaScript tutorials.

HTML Audio Element

The <audio> element embeds sound content in documents. It can contain multiple audio sources for different formats to ensure browser compatibility.

Basic Audio Syntax

Simple Audio Element:

<audio src="audio-file.mp3">
  Your browser does not support the audio element.
</audio>

Audio with Controls

Audio with Player Controls:

<audio controls>
  <source src="song.mp3" type="audio/mpeg">
  <source src="song.ogg" type="audio/ogg">
  <source src="song.wav" type="audio/wav">
  Your browser does not support the audio element.
</audio>

Audio Attributes

controls

Shows play/pause button, volume, progress bar

autoplay

Starts playing automatically (use carefully!)

loop

Repeats the audio when it ends

muted

Starts with audio muted

preload

How much to preload: "none", "metadata", or "auto"

Audio with Multiple Attributes:

<audio controls loop preload="metadata">
  <source src="background-music.mp3" type="audio/mpeg">
  <source src="background-music.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

<!-- Audio with autoplay (use sparingly) -->
<audio autoplay muted controls>
  <source src="intro-sound.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

HTML Video Element

The <video> element embeds video content in documents. Like audio, it supports multiple sources for different formats and includes many useful attributes.

Basic Video Syntax

Simple Video Element:

<video width="640" height="360" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

Video with Poster Image

Video with Thumbnail:

<video width="800" height="450" controls poster="video-thumbnail.jpg">
  <source src="presentation.mp4" type="video/mp4">
  <source src="presentation.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

Video Attributes

width & height

Set video dimensions in pixels

controls

Shows video player controls

poster

Image shown before video plays

autoplay

Starts playing automatically (often blocked by browsers)

loop

Repeats the video when it ends

muted

Starts with video muted

preload

How much to preload: "none", "metadata", or "auto"

Video with Multiple Attributes:

<!-- Responsive video -->
<video width="100%" height="auto" controls poster="thumb.jpg" preload="metadata">
  <source src="tutorial.mp4" type="video/mp4">
  <source src="tutorial.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

<!-- Looping background video (muted for autoplay) -->
<video width="1920" height="1080" autoplay muted loop>
  <source src="background-video.mp4" type="video/mp4">
  <source src="background-video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

HTML SVG Element

SVG (Scalable Vector Graphics) allows you to create vector graphics directly in HTML. Unlike canvas, SVG graphics are defined with markup and can be styled with CSS.

Basic SVG Syntax

Simple SVG Element:

<svg width="200" height="200">
  <circle cx="100" cy="100" r="50" fill="blue" />
</svg>

Live Example:

Blue circle created with SVG

SVG Basic Shapes

Common SVG Shapes:

<svg width="400" height="300">
  <!-- Rectangle -->
  <rect x="10" y="10" width="100" height="60" fill="red" />

  <!-- Circle -->
  <circle cx="200" cy="40" r="30" fill="green" />

  <!-- Ellipse -->
  <ellipse cx="320" cy="40" rx="40" ry="25" fill="purple" />

  <!-- Line -->
  <line x1="10" y1="100" x2="390" y2="100" stroke="black" stroke-width="2" />

  <!-- Polygon (triangle) -->
  <polygon points="50,150 100,250 0,250" fill="orange" />

  <!-- Polyline -->
  <polyline points="150,150 200,200 250,150 300,200 350,150"
            fill="none" stroke="blue" stroke-width="3" />

  <!-- Path (complex shape) -->
  <path d="M 10 280 Q 95 280 180 280" stroke="brown" stroke-width="3" fill="none" />
</svg>

SVG Text and Groups

SVG Text and Grouping:

<svg width="300" height="200">
  <!-- Text element -->
  <text x="50" y="30" font-family="Arial" font-size="20" fill="black">
    Hello SVG!
  </text>

  <!-- Group of elements -->
  <g fill="blue" stroke="black" stroke-width="2">
    <circle cx="100" cy="100" r="30" />
    <rect x="150" y="70" width="60" height="60" />
  </g>

  <!-- Text on a path -->
  <defs>
    <path id="textPath" d="M 20 150 Q 150 100 280 150" />
  </defs>
  <text font-size="14">
    <textPath href="#textPath">Text following a curved path</textPath>
  </text>
</svg>

Media File Formats

üéµ Audio Formats

  • MP3: Most widely supported
  • OGG: Open source, good quality
  • WAV: Uncompressed, large files
  • AAC: Good compression, Apple devices
  • FLAC: Lossless compression

üé¨ Video Formats

  • MP4: Most widely supported
  • WebM: Open source, web optimized
  • OGV: Open source alternative
  • AVI: Older format, large files
  • MOV: Apple QuickTime format

SVG Benefits

üìê Why Use SVG?

  • Scalable: Looks crisp at any size - perfect for responsive design
  • Small file sizes: Vector graphics are often smaller than bitmap images
  • Editable: Can be modified with HTML, CSS, and JavaScript
  • Accessible: Screen readers can understand SVG content
  • SEO friendly: Text in SVG is searchable by search engines
  • No pixelation: Always looks sharp on high-resolution displays
  • Interactive: Can respond to hover, click, and other events

Best Practices

‚úÖ Do

  • Provide multiple format sources for compatibility
  • Include fallback content for unsupported browsers
  • Use appropriate file formats for your content
  • Optimize file sizes for web delivery
  • Add meaningful alt text and descriptions
  • Test across different browsers and devices
  • Consider accessibility for all users

‚ùå Don't

  • Use autoplay for audio/video without user consent
  • Forget fallback content for older browsers
  • Use huge uncompressed media files
  • Rely solely on media for important information
  • Ignore accessibility considerations
  • Use canvas when SVG would be more appropriate
  • Forget to test on mobile devices

Complete Example

Multimedia Page Example:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Multimedia Showcase</title>
</head>
<body>
  <header>
    <h1>My Multimedia Page</h1>

    <!-- SVG Logo -->
    <svg width="100" height="50">
      <rect width="100" height="50" fill="blue" />
      <text x="50" y="30" text-anchor="middle" fill="white" font-size="16">LOGO</text>
    </svg>
  </header>

  <main>
    <section>
      <h2>Featured Video</h2>
      <video width="640" height="360" controls poster="video-thumb.jpg">
        <source src="featured-video.mp4" type="video/mp4">
        <source src="featured-video.webm" type="video/webm">
        Your browser does not support the video tag.
      </video>
    </section>

    <section>
      <h2>Background Music</h2>
      <audio controls>
        <source src="background.mp3" type="audio/mpeg">
        <source src="background.ogg" type="audio/ogg">
        Your browser does not support the audio element.
      </audio>
    </section>



    <section>
      <h2>Vector Illustration</h2>
      <svg width="300" height="200">
        <circle cx="150" cy="100" r="80" fill="lightblue" stroke="navy" stroke-width="3" />
        <text x="150" y="105" text-anchor="middle" font-size="18">SVG Circle</text>
      </svg>
    </section>
  </main>
</body>
</html>

Key Takeaways

  • HTML5 provides powerful multimedia elements: audio, video, and SVG
  • Always include multiple source formats for better browser compatibility
  • Use controls attribute to give users control over media playback
  • SVG is perfect for scalable vector graphics like icons and logos
  • Audio and video elements support multiple formats for cross-browser compatibility
  • Provide fallback content for browsers that don't support these elements
  • Consider accessibility and performance when using multimedia
  • Choose the right format and element for your specific use case
