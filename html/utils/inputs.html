HTML Input Types Explained

Explore the most common HTML input types and how they work.

1. Text Input

Used for short, single-line text input. Commonly used for names or usernames.

<input type="text" placeholder="Enter your name">

2. Email Input

Validates that the input is in email format (e.g. name@example.com).

<input type="email" placeholder="you@example.com">

3. Password Input

Masks the input for sensitive data like passwords.

<input type="password" placeholder="Enter password">

4. Number Input

Only accepts numerical input. You can set min and max values.

<input type="number" min="0" max="100">

5. Textarea

A multi-line text box, good for messages or descriptions.

<textarea rows="4"></textarea>

6. Checkbox

Lets the user select one or more options (on/off values).

<input type="checkbox"> Subscribe

7. Radio Buttons

Choose one option from a set.

<input type="radio" name="gender" value="male"> Male <input type="radio" name="gender" value="female"> Female

8. Submit Button

Sends form data to the server or triggers JavaScript.

<input type="submit" value="Submit">

9. Date Input

Provides a date picker for selecting dates.

<input type="date">

10. Time Input

Allows users to select a time (hours and minutes).

<input type="time">

11. DateTime Local Input

Combines date and time selection in one input.

<input type="datetime-local">

12. Range Slider

A slider control for selecting a value within a range.

<input type="range" min="0" max="100" value="50">

13. Color Picker

Opens a color picker for selecting colors.

<input type="color" value="#ff6600">

14. File Upload

Allows users to select and upload files from their device.

<input type="file">

15. URL Input

Validates that the input is a valid URL format.

<input type="url" placeholder="https://example.com">

16. Telephone Input

Optimized for phone number input, especially on mobile devices.

<input type="tel" placeholder="123-456-7890">

17. Search Input

Provides search functionality with a clear button on some browsers.

<input type="search" placeholder="Search...">

18. Select Dropdown

Creates a dropdown menu with multiple options to choose from.

<select> <option>Choose option</option> <option>Option 1</option> <option>Option 2</option> </select>

19. Hidden Input

Stores data that is not visible to users but sent with form submission.

<input type="hidden" name="user-id" value="12345">

Hidden inputs are not visible (this is just for demonstration)

20. Reset Button

Clears all form fields and resets them to their default values.

<input type="reset" value="Reset">

21. Button Element

A clickable button that can trigger JavaScript functions or form actions.

<button type="button">Click Me</button>

22. Week Input

Allows users to select a week and year.

<input type="week">

23. Month Input

Allows users to select a month and year.

<input type="month">

24. Image Button

Creates a clickable image that acts as a submit button.

<input type="image" src="button.png" alt="Submit" width="100" height="30">

25. Fieldset with Legend

Groups related form elements together with a descriptive legend.

<fieldset> <legend>Personal Information</legend> <input type="text" placeholder="Name"> </fieldset>
Personal Information

26. Datalist (Autocomplete)

Provides autocomplete suggestions for an input field.

<input list="browsers" placeholder="Choose browser"> <datalist id="browsers"> <option value="Chrome"> <option value="Firefox"> <option value="Safari"> </datalist>

27. Output Element

Displays the result of a calculation or user action.

<input type="range" id="slider" value="50"> <output for="slider">50</output>
50

28. Progress Bar

Shows the completion progress of a task.

<progress value="70" max="100">70%</progress> 70%

70% Complete

29. Meter Element

Displays a scalar measurement within a known range (gauge).

<meter value="6" min="0" max="10">6 out of 10</meter> 6 out of 10

Score: 6 out of 10

30. Option Groups in Select

Groups related options in a select dropdown for better organization.

<select> <optgroup label="Frontend"> <option>HTML</option> <option>CSS</option> </optgroup> <optgroup label="Backend"> <option>Node.js</option> <option>Python</option> </optgroup> </select>
